vibe coding: 人机协作下的新型编程模式

2026-03-24
516 字 2 分钟阅读 阅读量: 0

最近想利用vibe coding对科研想法进行验证，多范式代码下的SQL修正，企图验证大模型对不同语言的知识储备是否可以启发帮助SQL的修正。

完全依赖vide coding：
代码可以运行，但是运行效率很低，且过程完全黑盒化。结果不尽人意。
试图重构，发下以下问题：

  • 知识老旧： 大模型无法遵循日益迅速发展的框架的开发习惯，表现在我试图在rule中让大模遵循使用langgraph开发，并使用context7查看最新文档进行开发。没有遵循的原因可能是大模型训练语料中未能及时将最新的数据纳入；更重要的可能是我并未对任务足够的细粒度拆分，导致大模型注意力分散。
  • 项目过于复杂，开发出屎山代码，难以维护：进行debug熟悉项目，发现代码结构混乱，且由于让大模型重构多次，历史遗留代码，重构不充分的问题暴露无遗，项目难以继续开发。

人机交互模式：
人工设计+ai功能代码撰写：抛弃完全依赖模型设计与开发的模式，改为以人为主，大模型功能实现的模式。
为什么这么做呢？
我发现如果大模型写好计划并按照任务执行，计划赶不上变化，同时我无法了解代码为什么这样设计，是否经充分的验证。所以完全由ai撰写的模式适用于demo的展示，针对复杂项目，更依赖的是以人为主体，ai加快代码撰写的人机交互模式。

code agent发展至今，全局记忆，统筹调度的能力还有待提高，期望有一天，记忆可以内化，资源可以充分调度，也非常乐意完全当一名产品经理。
